Introduction¶

  • So far we have considered a few types of graphs
    • Undirected graph: nodes AAA and BBB are connected by an edge
    • Directed graph: connection from node AAA to node BBB
    • Strong/weak graphs: each edge is labeled as strong or weak
  • Today we extend our understanding of networks to talk about weighted graphs
    • Each edge is assigned a float denoting the strength of tie
    • Ties can be positive (friends) or negative (enemies)
    • Can also very in strength (+2.0 better friends than +0.2)

Weighted Adjacency Matrix¶

  • In a simple (unweighted) graph, we used a matrix of 0’s and 1’s as an adjacency matrix
    • A 1 in row i column j marked an edge between i and j (or from i->j for directed)
    • A 0 marked lack of an edge

Shortest Paths¶

  • We talked previously about shortest paths for a Graph
  • This was defined as the minimum number of edges needed to move from node n1 to node n2
  • When we have a weighted graph things get more interesting...
  • Let wabw_{ab}wab​ represent the weight connecting nodes AAA and BBB
  • Define the shortest path between n1 and n2 as the path that minimizes ∑wab\sum w_{ab}∑wab​ for all edges A->B along a path

  • Let J(v)J(v)J(v) be the minimum cost-to-go from node vvv to node G

  • With J(v)J(v)J(v) in hand, the following algorithm will find the cost-minimizing path from AAA to GGG:
    1. Start with v=Av = Av=A
    2. From current node vvv move to any node that solves min⁡n∈Fvwvn+J(n)\min_{n \in F_v} w_{vn} + J(n)minn∈Fv​​wvn​+J(n), where FvF_vFv​ is the set of nodes that can be reached from vvv.
    3. Update notation to set v=nv = nv=n
    4. Repeat steps 2-3 (making note of which we visit) until v=Gv = Gv=G

But what about J(v)J(v)J(v)¶

  • The shortest path algorithm we presented above sounds simple, but assumed we know J(v)J(v)J(v)
  • How can we find it?
  • If you stare at the following equation long enough, you’ll be convinced that JJJ satisfies
    J(v)=min⁡n∈Fvwvn+J(n)J(v) = \min_{n \in F_v} w_{vn} + J(n)J(v)=n∈Fv​min​wvn​+J(n)

  • This is known as the Bellman equation
  • It is a restriction that JJJ must satisfy
  • We’ll use this restriction to compute JJJ

Computing J: Guess and Iterate¶

  • We’ll present the standard algorithm for computing J(v)J(v)J(v)
  • This is an iterative method
  • Let iii represent the iteration we are on and Ji(v)J_i(v)Ji​(v) be the guess for J(v)J(v)J(v) on iteration iii

function compute_J(G::SimpleWeightedDiGraph, dest_node::Int)
    N = nv(G)
    # step 1. start with zeros
    i = 0
    Ji = zeros(N)

    next_J = zeros(N)

    W = weights(G)

    done = false
    while !done
        i += 1
        for v in 1:N
            if v == dest_node
                next_J[v] = 0
                continue
            end
            F_v = neighbors(G, v)
            costs = [cost(W, Ji, n, v) for n in F_v]
            next_J[v] = minimum(costs)
        end
        done = all(next_J .≈ Ji)
        copy!(Ji, next_J)
    end
    Ji
end

Summary¶

  • Weighted graphs allow us to analyze the cost of travsersing paths
    • Applied in situations like traffic flows (on physical roads/bridges), resource planning, supply chain, international trade (weights as tarrifs), and more
  • Programming skills...
