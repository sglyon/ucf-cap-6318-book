Tabular Q – review¶

  • Previous examples (farkle, career choice) have had discrete state and action spaces
    • Career choice:
      S={ϵi}i=1N×{θi}i=1NA={ stay put, new job, new life}\begin{aligned} \mathcal{S} &= \{\epsilon_i\}_{i=1}^N \times \{\theta_i\}_{i=1}^N \\ \mathcal{A} &= \{\text{ stay put}, \text{ new job}, \text{ new life}\} \end{aligned}SA​={ϵi​}i=1N​×{θi​}i=1N​={ stay put, new job, new life}​
      (1)
    • Farkle:
      S=round×scores×can roll×rolled×turn sumA(s)⊆roll×stop×play dice\begin{aligned} \mathcal{S} &= \text{round} \times \text{scores} \times \text{can roll} \times \text{rolled} \times \text{turn sum} \\ \mathcal{A}(s) &\subseteq \text{roll} \times \text{stop} \times \text{play dice} \end{aligned}SA(s)​=round×scores×can roll×rolled×turn sum⊆roll×stop×play dice​
      (2)
  • This allowed us to use a lookup table for storing Q(s,a)Q(s, a)Q(s,a)
    • Q was a dict, each key was (s,a)(s, a)(s,a) pair and each value was associated Q

Tabular Q – limitations¶

  • Not all environments or RL problems have discrete state and action spaces
  • Some have much larger state or action spaces
    • Self driving car: all possible images, sensor readings
    • Stock trading: sequences of prices, liquidity, funds
    • Medical imaging: all possible medical images
  • Using a lookup table (dict) to represent QQQ in these cases is not feasible
  • We need a generalization strategy for QQQ
  • Our response: function approximation

Linear Function Approximation¶

  • Let’s begin our generalizing Q with linear function approximation
  • Let x(s):S→RDx(s): \mathcal{S} \rightarrow \mathbb{R}^Dx(s):S→RD be a feature map for state sss
  • xxx could be identity function, polynomial features, spline basis functions, etc
  • For the discrete action space case we will let wa∈RDw_a \in \mathbb{R}^Dwa​∈RD represent weights such that
    Q(s,a∣w)=waTx(s)Q(s, a | w) = w_a^Tx(s)Q(s,a∣w)=waT​x(s)
    (3)
  • Notice that our approximation to QQQ is linear in the features x(s)x(s)x(s)

SGD¶

  • We would like to choose weights waw_awa​ that minimize the difference between q∗(s,a)q^*(s,a)q∗(s,a) and Q(s,a∣wa)Q(s,a|w_a)Q(s,a∣wa​)
  • To do this we will use the MSE loss function between q∗(s,a)q^*(s,a)q∗(s,a) and Q(s,a∣wa)Q(s,a|w_a)Q(s,a∣wa​)
  • If we apply SGD using this loss function, we want to update www as follows:
    wA=wA−12α∇w[q∗(S,A)−Q(S,A∣w)]2=wA+α[q∗(S,A)−Q(S,A∣w)]∇wQ(S,A∣w)\begin{aligned} w_A &= w_A - \frac{1}{2} \alpha \nabla_w \big[q^*(S,A) - Q(S, A | w) \big]^2 \\ &= w_A + \alpha \big[q^*(S, A) - Q(S, A | w) \big] \nabla_w Q(S, A | w) \end{aligned}wA​​=wA​−21​α∇w​[q∗(S,A)−Q(S,A∣w)]2=wA​+α[q∗(S,A)−Q(S,A∣w)]∇w​Q(S,A∣w)​
    (4)
  • The issue is that we don’t know q∗(S,A)q^*(S, A)q∗(S,A)...

TD(0) to the rescue¶

  • Notice how the update rule for wAw_AwA​ looks very similar to the update rule for tabular Q-learning:
    wA=wA+α[q∗(S,A)−Q(S,A∣w)]∇wQ(S,A∣w)Q(S,A)=Q(S,A)+α[R+βmax⁡a′Q(S′,a′)−Q(S,A)]\begin{aligned} w_A &= w_A + \alpha \big[q^*(S, A) - Q(S, A | w) \big] \nabla_w Q(S, A | w) \\ Q(S, A) &= Q(S, A) + \alpha \big[R + \beta \max_{a'} Q(S', a') - Q(S, A) \big] \end{aligned}wA​Q(S,A)​=wA​+α[q∗(S,A)−Q(S,A∣w)]∇w​Q(S,A∣w)=Q(S,A)+α[R+βa′max​Q(S′,a′)−Q(S,A)]​
    (5)
  • In Q-learning we take small steps in the direction of q∗(s,a)q^*(s, a)q∗(s,a) using the TD(0) error
  • We’ll extend this knowledge to the SGD for wAw_AwA​ and use R+βmax⁡a′Q(S′,a′∣w)R + \beta \max_{a'} Q(S', a'| w)R+βmaxa′​Q(S′,a′∣w) in place of q∗(S,A)q^*(S, A)q∗(S,A):
    wA=wA+α[R+βmax⁡a′Q(S′,a′∣w)−Q(S,A∣w)]∇wQ(S,A∣w)w_A = w_A + \alpha \big[R + \beta \max_{a'} Q(S', a'| w) - Q(S, A | w) \big] \nabla_w Q(S, A | w)wA​=wA​+α[R+βa′max​Q(S′,a′∣w)−Q(S,A∣w)]∇w​Q(S,A∣w)
    (6)
  • There are proofs that provide guarantees that if we use this update rule Q(s,a∣w)Q(s, a | w)Q(s,a∣w) will converge to q∗q^*q∗
  • We will take these proofs as given

SGD + TD(0) + Linear Approximator¶

  • We now combine the concepts of SGD, TD(0) learning, and the fact that we consider forms of Q(s,a∣w)=waTx(s)Q(s, a | w) = w_a^Tx(s)Q(s,a∣w)=waT​x(s) that are linear in features x(s)x(s)x(s)
  • Under our linearity assumption
    ∇wQ(s,a∣w)=x(s)\nabla_w Q(s, a | w) = x(s)∇w​Q(s,a∣w)=x(s)
    (7)
  • This makes our update rule for www equal to:
    wA=wA+α[R+βmax⁡a′Q(S′,a′∣w)−Q(S,A∣w)]x(s)w_A = w_A + \alpha \big[R + \beta \max_{a'} Q(S', a'| w) - Q(S, A | w) \big] x(s)wA​=wA​+α[R+βa′max​Q(S′,a′∣w)−Q(S,A∣w)]x(s)
    (8)
  • Note that the TD(0) part of the update rule is a scalar
  • We update our weights in the direction of our feature vector x(s)x(s)x(s), with scaled step size according to TD(0) term and learning rate

Example: Complete Polynomials¶

  • Let’s try this procedure out using a function x(s)x(s)x(s) that computes a complete polynomial in sss
  • A complete polynomial of degree DDD is composed of all possible multiplicative terms of the elements sss that are of total degree less than or equal to DDD
    • Example: s=[a,b,c]s = [a, b, c]s=[a,b,c] and D=2D = 2D=2:
      x(s)=[1abca2abacb2bcc2]Tx(s) = \begin{bmatrix}1 & a & b & c & a^2 & a b & a c & b^2 & b c & c^2 \end{bmatrix}^Tx(s)=[1​a​b​c​a2​ab​ac​b2​bc​c2​]T
      (9)
    • Does not include a2ba^2 ba2b term because that has total degree of 3>23 > 23>2

  • We’ll now create a class that uses complete_polynomial under the hood, and keeps track of w for us
class CompletePolyInterpolator:
    def __init__(self, N: int, D: int=2):
        self.D = D
        self.w = np.random.rand(complete_polynomial(np.zeros(N), D).size)

    def basis(self, s):
        "return basis matrix x(s)"
        return complete_polynomial(s, self.D)

    def __call__(self, s):
        return self.basis(s) @ self.w
  • We’ll wrap this in a class that stores one CompletePolyInterpolator per discrete action
class ContinuousSDiscreteAQ:
    def __init__(self,  Ndim_s: int, Na:int, approximator = CompletePolyInterpolator):
        self.Q = [approximator(Ndim_s) for _ in range(Na)]
        self.Na = Na

    @property
    def w(self):
        return np.row_stack([Qa.w for Qa in self.Q])

    def __call__(self, s, a):
        return self.Q[a](s)

    def __getitem__(self, a):
        return self.Q[a]

    def basis(self, s, a):
        return self.Q[a].basis(s)

    def get_greedy(self, s, A_s):
        vals = [self(s, a) for a in A_s]
        max_val = max(vals)
        return random.choice([a for (a, v) in zip(A_s, vals) if v == max_val])
  • And now we’ll create a Qlearning class that can leverage it
class ApproxQlearning(object):
    def __init__(
            self,
            environment,
            approximator=CompletePolyInterpolator,
            epsilon=0.9, alpha=0.05, beta=0.5
        ):
        self.env = environment
        Ns = env.observation_space.shape[0]

        assert isinstance(env.action_space, gym.spaces.Discrete)
        Na = env.action_space.n
        self.A_s = np.arange(Na)
        self.Q = ContinuousSDiscreteAQ(Ns, Na, approximator=approximator)
        self.epsilon = epsilon
        self.alpha = alpha
        self.beta = beta

        self.restart_episode()

    def restart_episode(self):
        self.s, _ = self.env.reset()

    def get_greedy(self, s):
        return self.Q.get_greedy(s, self.A_s)

    def generate_A(self, s):
        if random.random() > self.epsilon:
            return self.env.action_space.sample()
        return self.get_greedy(s)

    def step(self):
        s = self.s

        # first generate an A
        a = self.generate_A(s)

        # take step
        sp, r, done, _truncated, _info = self.env.step(a)

        if done:
            # game is over
            self.s = sp
            return done

        # get greedy a' based on Q and sp
        ap = self.get_greedy(sp)

        # Do TD update
        Q, α, β = self.Q, self.alpha, self.beta  # simplify notation
        Q[a].w = Q[a].w + α * (r + β * Q(sp, ap) - Q(s, a)) * Q[a].basis(s)

        # step forward in time
        self.s = sp
  • We’ll train qlearning for 5,000 steps and again plot the average number of steps per game
ql = ApproxQlearning(env, epsilon=0.98, beta=0.8, alpha=0.05)

Summary¶

  • Our complete polynomial approximation seems to do better than random
  • This is a good start, but this has its limits:
    • Only way to add more flexibility to our approximator is increase polynomial degree
    • The min/max values for the velocity variables are extreme
    • Trying to compute quadratic or greater terms in very large values will likely cause numerical instability
  • We need a different paradigm for representing QQQ in a more flexible, but numerically stable way
