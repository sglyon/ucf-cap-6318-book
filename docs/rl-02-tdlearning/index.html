Reminder: Dynamic programming¶

  • Let’s begin by recalling what we know about dynamic programming
  • Recall the cake eating problem:
    • Time is discrete
    • β\betaβ is discount factor
    • Size of cake is xˉ\bar{x}xˉ
    • Consumption of cake in period ttt is ctc_tct​
    • Utility function u:R→Ru: \mathbb{R} \rightarrow \mathbb{R}u:R→R maps from consumption today into happiness

Sequential Problem¶

  • Objective
    max⁡ct∑t=0∞βtu(ct)subject to ∑t=0∞ct≤xˉct≥0∀t\begin{aligned} \max_{c_t} &\sum_{t=0}^{\infty} \beta^t u(c_t) \\ \text{subject to } \quad & \sum_{t=0}^{\infty} c_t \le \bar{x} \\ & c_t \ge 0 \quad \forall t \end{aligned}ct​max​subject to ​t=0∑∞​βtu(ct​)t=0∑∞​ct​≤xˉct​≥0∀t​
    (1)
  • Need to solve for infinite sequence {ct}t\{c_t\}_t{ct​}t​
  • Or...

Value Function¶

  • We can set up a value function
    v(xˉ)≡∑t=0∞βtu(ct)v(\bar{x}) \equiv \sum_{t=0}^{\infty} \beta^t u(c_t)v(xˉ)≡t=0∑∞​βtu(ct​)
    (2)
  • v(xˉ)v(\bar{x})v(xˉ) is the total value the consumer places on having a cake of size xˉ\bar{x}xˉ
  • Decompose vvv into two steps: first period + later periods
    v(xˉ)=u(c0)⏟flow utility+β∑t=1∞βt−1u(ct)⏟continuation utilityv(\bar{x}) = \underbrace{u(c_0)}_{\text{flow utility}} + \underbrace{\beta \sum_{t=1}^{\infty} \beta^{t-1} u(c_t)}_{\text{continuation utility}}v(xˉ)=flow utilityu(c0​)​​+continuation utilityβt=1∑∞​βt−1u(ct​)​​
    (3)

Recursive Formulation¶

  • Note: continuation utility depends on xt+1=xt−ctx_{t+1} = x_t - c_txt+1​=xt​−ct​
  • Use this observation to write v:R+→Rv: \mathbb{R}^+ \rightarrow \mathbb{R}v:R+→R recursively:
    v(xt)=max⁡0≤ct≤xu(ct)⏟flow utility+βv(xt−ct)⏟continuation value\begin{aligned} v(x_t) &= \max_{0 \leq c_t \leq x} \underbrace{u(c_t)}_{\text{flow utility}} + \underbrace{\beta v(x_t - c_t)}_{\text{continuation value}} \end{aligned}v(xt​)​=0≤ct​≤xmax​flow utilityu(ct​)​​+continuation valueβv(xt​−ct​)​​​
    (4)
  • This is known as the Bellman Equation

Solution to Recursive Problem¶

  • A solution to the dynamic program consists of two functions:
    1. Value function v∗(x):R+→Rv^*(x): \mathbb{R}^+ \rightarrow \mathbb{R}v∗(x):R+→R -- value of beginning period with xxx cake remaining
    2. Policy function c∗(x):R+→[0,x]c^*(x): \mathbb{R}^+ \rightarrow [0, x]c∗(x):R+→[0,x] -- optimal level of consumption with xxx cake remaining
  • Under certain regularity conditions (which we assume), the recursive problem (and its solution) is equivalent to the sequential problem we started with

Connection to RL¶

  • How does this connect to RL?
  • (S, A, R) pattern for RL is very closely related to recursive formulation of dynamic programming
    • St⟹xtS_t \Longrightarrow x_tSt​⟹xt​
    • At⟹ctA_t \Longrightarrow c_tAt​⟹ct​
    • Rt⟹u(ct)R_t \Longrightarrow u(c_t)Rt​⟹u(ct​)
  • Expressing v(x)=flow utility+ continuation valuev(x) = \text{flow utility} + \text{ continuation value}v(x)=flow utility+ continuation value is like repeating (S, A, R) sequence many times
  • Baseline algorithm for solving DP problem (VFI) is quite similar to how basic RL algorithms work
    • Start with guess for value, make decision, update guess, repeat...

Baseline Assumptions¶

  • Let state space S\mathcal{S}S and action space A\mathcal{A}A be discrete

  • Let At∈A(St)A_t \in \mathcal{A}(S_t)At​∈A(St​) represent action at time ttt

State Value Function¶

  • Let v∗(s)v^*(s)v∗(s) be the optimal value of being in state sss (called state value function)
  • We write this as:
    v∗(s)=max⁡a∈A(s)E[Rt+1+βv∗(St+1)∣St=s,At=a]=max⁡a∈A(s)∑s′,rp(s′,r∣s,a)[r+βv∗(s′)]\begin{aligned} v^*(s) &= \max_{a \in \mathcal{A}(s)} E \left[R_{t+1} + \beta v^*(S_{t+1}) | S_t=s, A_t=a \right] \\ &= \max_{a \in \mathcal{A}(s)} \sum_{s', r} p(s', r | s, a) \left[ r + \beta v^*(s') \right] \end{aligned}v∗(s)​=a∈A(s)max​E[Rt+1​+βv∗(St+1​)∣St​=s,At​=a]=a∈A(s)max​s′,r∑​p(s′,r∣s,a)[r+βv∗(s′)]​
    (6)
  • Should be familiar from our dynamic programming studies
  • Note expectation around the flow utility term Rt+1R_{t+1}Rt+1​, leaving room for that reward to be stochastic

Action Value Function¶

  • We can also write an action value function
  • Let q∗(s,a)q^*(s, a)q∗(s,a) be the optimal value of being in state sss and choosing action aaa:
    q∗(s,a)=E[Rt+1+βmax⁡a′∈A(St+1)q∗(St+1,a′)∣St=s,At=a]=∑s′,rp(s′,r∣s,a)[r+βmax⁡a′∈A(St+1)q∗(s′,a′)]\begin{aligned} q^*(s, a) &= E \left[R_{t+1} + \beta \max_{a' \in \mathcal{A}(S_{t+1})} q^*(S_{t+1}, a') | S_t=s, A_t=a \right] \\ &= \sum_{s', r} p(s', r | s, a) \left[ r + \beta \max_{a' \in \mathcal{A}(S_{t+1})} q^*(s', a') \right] \end{aligned}q∗(s,a)​=E[Rt+1​+βa′∈A(St+1​)max​q∗(St+1​,a′)∣St​=s,At​=a]=s′,r∑​p(s′,r∣s,a)[r+βa′∈A(St+1​)max​q∗(s′,a′)]​
    (7)

  • The function q∗(s,a)q^*(s, a)q∗(s,a) is more general than v∗(s)v^*(s)v∗(s):
    v∗(s)=max⁡a∈A(s)q∗(s,a)v^*(s) = \max_{a \in \mathcal{A}(s)} q^*(s, a)v∗(s)=a∈A(s)max​q∗(s,a)

Acting with q∗q^*q∗¶

  • Goal of RL is to learn to make decisions that maximize ∑βtRt\sum \beta^t R_t∑βtRt​
  • Knowing q∗(s,a)q^*(s, a)q∗(s,a) tells us maximium value of being in state sss and choosing aaa
  • If we knew q∗q^*q∗, acting optimally would be easy:
    a∗(s)=argmaxa∈A(s)q∗(s,a)a^*(s) = \text{argmax}_{a \in \mathcal{A}(s)} q^*(s, a)a∗(s)=argmaxa∈A(s)​q∗(s,a)
    (9)
  • However, we rarely if ever know q∗q^*q∗, so we must approximate it
  • We will let Q(s,a)Q(s, a)Q(s,a) represent our approximation of q∗q^*q∗

Approximating q∗q^*q∗¶

  • The goal of TD learning is to find an accurate approximation Q(s,a)Q(s, a)Q(s,a) such that Q(s,a)≈q∗(s,a)  ∀s,a)Q(s, a) \approx q^*(s, a) \; \forall s, a)Q(s,a)≈q∗(s,a)∀s,a)
  • There are many RL algorithms that seek to do this
  • We’ll focus on two:
    • Sarsa
    • Q-learning

Temporal Differences¶

  • The Bellman equaition for our approximation Q(s,a)Q(s, a)Q(s,a) is
    Q(s,a)=E[R′+βmax⁡a′Q(s′,a′)∣s,a]Q(s,a) = E[R' + \beta \max_{a'} Q(s', a') | s,a]Q(s,a)=E[R′+βa′max​Q(s′,a′)∣s,a]
    (10)
  • Suppose that we interacted with environment and have in hand (St,At,Rt+1,St+1,At+1)(S_t, A_t, R_{t+1}, S_{t+1}, A_{t+1})(St​,At​,Rt+1​,St+1​,At+1​)
  • Now we plug these into the Bellman by:
    1. Using the form of the Bellman
    2. But drop EEE and max⁡\maxmax because we already know the transition that did occur from (St,At,Rt+1,St+1,At+1)(S_t, A_t, R_{t+1}, S_{t+1}, A_{t+1})(St​,At​,Rt+1​,St+1​,At+1​)
    3. Change === an ≈\approx≈ because this isn’t evaluating full Bellman
      Q(St,At)≈Rt+1+βQ(St+1,At+1)Q(S_t,A_t) \approx R_{t+1} + \beta Q(S_{t+1}, A_{t+1})Q(St​,At​)≈Rt+1​+βQ(St+1​,At+1​)
      (11)
  • The difference between the left and right and sides is known as a temporal difference:
    TD(0)(Q)≡Rt+1+βQ(St+1,At+1)−Q(St,At)TD(0)(Q) \equiv R_{t+1} + \beta Q(S_{t+1}, A_{t+1}) - Q(S_t, A_t)TD(0)(Q)≡Rt+1​+βQ(St+1​,At+1​)−Q(St​,At​)
    (12)

There are extensions to the temporal difference that allow for multiple time periods. The 0 in TD(0)TD(0)TD(0) indicates that this is one-step TD learning. See Chapters 7 and 12 of Sutton/Barto for more info

Learning using TD(0)¶

  • We can use temporal differences to improve our approximation QQQ:
  • Let Qt(s,a)Q_t(s, a)Qt​(s,a) represent our approximation at the start of period ttt
  • Similar to gradient descent methods, we will take a step from QtQ_tQt​ in a direction that improves its accuracy
  • To improve accuracy we step in direction of TD(0)(Qt)TD(0)(Q_t)TD(0)(Qt​) (using step size α\alphaα):
    Qt+1(St,At)=Qt(St,At)+αTD(0)(Qt)=Qt(St,At)+α[Rt+1+βQt(St+1,At+1)−Qt(St,At)]\begin{aligned} Q_{t+1}(S_t, A_t) &= Q_t(S_t, A_t) + \alpha TD(0)(Q_t) \\ &= Q_t(S_t, A_t) + \alpha \left[R_{t+1} + \beta Q_t(S_{t+1}, A_{t+1}) - Q_t(S_t, A_t) \right] \end{aligned}Qt+1​(St​,At​)​=Qt​(St​,At​)+αTD(0)(Qt​)=Qt​(St​,At​)+α[Rt+1​+βQt​(St+1​,At+1​)−Qt​(St​,At​)]​
    (13)

Sarsa Algorithm¶

  • The Sarsa algorithm applies the update rule we just described
  • The algorithm is summarized by Barto and Sutton as follows (section 6.4)
from collections import defaultdict
import random

import numpy as np
import matplotlib.pyplot as plt
class TabularQ:
    def __init__(self):
        self.Q = defaultdict(lambda: 0)

    def __call__(self, s, a):
        return self.Q[(s.observable_state(), a)]

    def __setitem__(self, k, v):
        s, a = k
        self.Q[(s.observable_state(), a)] = v

    def get_greedy(self, s, A_s):
        vals = [self(s, a) for a in A_s]
        max_val = max(vals)
        return random.choice([a for (a, v) in zip(A_s, vals) if v == max_val])
class Sarsa(object):
    def __init__(self, environment, epsilon=0.9, alpha=0.1, beta=1.0):
        self.env = environment
        self.Q = TabularQ()
        self.epsilon = epsilon
        self.alpha = alpha
        self.beta = beta

        self.restart_episode()

    def restart_episode(self):
        # These will be (S, A) in our notation. Need to initialize
        self.s = self.env.reset()
        self.a = self.act(self.s, self.env.enumerate_options(self.s))

    def get_greedy(self, s, A_s):
        return self.Q.get_greedy(s, A_s)

    def act(self, s, A_s):
        if random.random() > self.epsilon:
            return random.choice(A_s)
        return self.get_greedy(s, A_s)

    def done(self, s=None) -> bool:
        return self.env.done(s if s else self.s)

    def step(self):
        # first take the step (s, a)
        s, a = self.s, self.a
        sp, r = self.env.step(s, a)

        if self.done(sp):
            # game is over
            self.s = sp
            return

        # then use policy to compute ap
        A_sp = self.env.enumerate_options(sp)
        ap = self.act(sp, A_sp)

        # now we know S-A-R-S'-A' -- ready to do update
        Q, α, β = self.Q, self.alpha, self.beta  # simplify notation
        Q[(s, a)] = Q(s, a) + α * (r + β * Q(sp, ap) - Q(s, a))

        # step forward in time
        self.s = sp
        self.a = ap

Farkle env¶

  • Let’s wrap farkle code into environment Sarsa expects
  • Need a few key methods:
    • reset() -> State
    • enumerate_options(state) -> List[Action]
    • step(s: State, a: Action) -> Tuple[State, Reward]
    • done(s:State) -> bool
from farkle import State, Action, RandomFarklePlayer, FarklePlayer, STOP, BANKRUPT
from typing import List, Tuple

class FarkleEnv:
    # first, some helper methods
    def __init__(
            self,
            opponent: FarklePlayer=RandomFarklePlayer(),
            points_to_win=10_000,
            verbose: bool = False
        ):
        self.points_to_win = points_to_win
        self.opponent = opponent
        self.n_players = 2
        self._state = State(self.n_players)
        self._history: List[Tuple[State, Action]] = []

    @property
    def state(self) -> State:
        return self._state

    def set_state(self, action: Action, new_state: State):
        self._history.append((self.state, action))
        self._state = new_state

    def opponent_turn(self, s: State) -> State:
        choices = s.enumerate_options()
        action = self.opponent.act(s, choices)
        sp = s.step(action)

        # check if player chose to stop
        if sp.current_player != 1:
            return sp

        # Player didn't stop, but still their turn. Call again
        return self.opponent_turn(sp)

    # key methods needed
    def done(self, state) -> bool:
        return any(score > self.points_to_win for score in state.scores)

    def reset(self):
        self._state = State(self.n_players)
        self._history = []
        return self.state.roll()

    def step(self, s: State, a: Action) -> Tuple[State, int]:
        sp = s.step(a)
        r = 0

        # see if we ended
        if sp.current_player != 0:
            if a is STOP:
                # only score when we choose to stop
                r = s.turn_sum

            # take opponent turn
            sp = self.opponent_turn(sp)

        self.set_state(a, sp)
        return sp, r

    def enumerate_options(self, s: State) -> List[Action]:
        return self.state.enumerate_options(s.rolled_dice)

%%time

def play_many_games(N):
    terminal_states = []
    print_skip = N // 10
    for i in range(N):
        play_game(sarsa)
        terminal_states.append(sarsa.s)
        if i % print_skip == 0:
            print(f"Done with {i}/{N} (len(Q) = {len(sarsa.Q.Q)})")
    return terminal_states
